# Chapter 01: Introduction

Currently, consumers require the ability to quickly search for products,
process orders, and have accurate stock control. As a result, people
should evaluate the status of the inventory, demand, sales potential,
possible problems with the stocks, and others within a minimum amount of
time without spending too much effort on working with numbers manually.
Thus, Business Intelligence (BI) systems become indispensable for such
tasks, even in small projects.

The initial project aimed at using machine learning for demand
management, a data-driven application will be developed that will be
used to manage customers' orders, demand signals, inventories, alerts,
and others related to retail products in a web application similar to
that provided by Amazon.

There are multiple reasons why the project will be practical. Firstly,
both groups of stakeholders, namely customers and workers, are included.
Secondly, all business information is presented visually, so users can
work more effectively. In general, all features listed make the current
project a good fit for using BI because of data cleansing,
visualization, KPIs, inventories, transactions, and decision making.

The main goal of the project cannot be limited to creating an elegant
interface but should focus on producing a product that helps to convert
data into decisions to be made by people. For instance, notifications
will be issued when there is not enough inventory in stock. Products
will be sorted out using ABC analysis, and Economic Order Quantity will
be calculated.
